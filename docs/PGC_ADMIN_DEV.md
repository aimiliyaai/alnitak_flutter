# PGC 管理前端开发文档

本文档描述 PGC（专业/合作内容）管理前端的范围、功能与实现约定，供后续开发使用。管理端为独立前端（Web），与现有 Flutter 用户端分离；对接现有 Go 后端 PGC 相关 API。

---

## 1. 定位与技术栈建议

- **定位**：运营/创作者使用的 Web 管理后台，负责 PGC 的创建、编辑、剧集管理、发布等。
- **技术栈建议**：**Vue 3 + Vite + Element Plus**（或 Naive UI）；构建产物可部署至现有 Go 服务的 `static/admin`，通过 `/admin` 访问。
- **鉴权**：请求后端 API 时携带 Token（如 `Authorization: Bearer <token>`），与现有后端鉴权一致；前端提供 Token 输入或登录页，可选「记住 Token」。

---

## 2. 功能范围

### 2.1 核心功能（首版必做）

| 功能 | 说明 | 备注 |
|------|------|------|
| **PGC 列表** | 分页列表；按类型筛选（番剧/纪录片/电影/电视剧）；按状态筛选（若后端支持）；支持标题/关键词搜索 | 与后端 Search/List 接口对接 |
| **创建 PGC** | 表单：类型、标题、封面图（URL 或上传）、简介、年份、地区、评分、是否连载；剧集表至少一集，每集必填站内视频 ID（vid） | 创建即通过（或按后端 status 约定） |
| **编辑 PGC** | 修改元数据（类型、标题、封面、简介等）；剧集在「剧集管理」页单独维护 | 不在此表单内直接增删剧集 |
| **剧集管理** | 进入某 PGC 后：查看剧集列表、添加剧集（集序 + 标题 + vid）、删除剧集；若后端支持则支持排序 | 对应 `/:pgc_id/episodes` 相关 API |
| **删除 PGC** | 删除整条 PGC，带二次确认；软删/硬删与后端一致 | 对应 DELETE `/:pgc_id` |
| **登录/鉴权** | Token 输入或登录页；所有请求 Header 带 Token；可选「记住 Token」 | 与现有 API 鉴权一致 |

### 2.2 扩展功能（常用，按需排期）

| 功能 | 说明 |
|------|------|
| **封面/图片上传** | 创建/编辑时上传封面图，不再仅支持 URL；需后端提供上传接口或复用现有文件服务 |
| **草稿 / 发布状态** | 支持草稿、提交审核、通过后发布；依赖后端状态字段与审核接口 |
| **批量操作** | 批量删除、批量修改类型/状态等（若业务需要） |
| **预览** | 保存前预览卡片在列表/详情页的展示效果 |
| **剧集顺序** | 拖拽排序或排序号，并同步到后端（若后端支持 order 字段） |

### 2.3 可选/后续功能

| 功能 | 说明 |
|------|------|
| **数据统计** | 单条 PGC 的播放量、完播率等（需后端统计与接口） |
| **评论/弹幕管理** | 对 PGC 下评论/弹幕的隐藏、删除等（若产品需要） |
| **权限与角色** | 不同角色（运营/审核/超管）可见菜单与操作不同 |
| **操作日志** | 记录创建/编辑/删除/发布操作与操作人，便于审计 |
| **关联推荐** | 配置「相关推荐」列表（若产品有该需求） |

## 3. 页面与路由建议

| 路径/页面 | 功能 |
|-----------|------|
| `/` 或 `/pgc` | PGC 列表（筛选、搜索、分页）；操作：新建、编辑、剧集、删除 |
| `/pgc/create` | 新建 PGC 表单（元数据 + 剧集表，至少一集） |
| `/pgc/:id/edit` | 编辑 PGC 元数据 |
| `/pgc/:id/episodes` | 剧集管理：列表、添加剧集、删除剧集、排序（若支持） |
| `/login` 或 弹窗 | Token 输入/登录（若采用独立登录页） |

列表页表格列建议：序号、封面缩略图、标题、类型、集数、状态、创建时间、操作（编辑、剧集、删除）。

## 4. 与现有后端 API 的对应关系

以下为现有 Go 后端 PGC 相关接口的约定，前端按此对接即可。

| 前端能力 | 方法 | 路径/说明 |
|----------|------|-----------|
| 列表/搜索 | GET | `/api/v1/pgc`，参数：type、keyword、page、page_size 等 |
| 详情（含剧集） | GET | `/api/v1/pgc/:pgc_id` 或 detail-with-episodes 类接口 |
| 创建 | POST | `/api/v1/pgc`，Body：类型、标题、封面、简介、剧集列表等 |
| 更新 | PUT | `/api/v1/pgc/:pgc_id`，Body：元数据 |
| 删除 | DELETE | `/api/v1/pgc/:pgc_id` |
| 剧集列表 | GET | `/api/v1/pgc/:pgc_id/episodes`，支持分页 |
| 添加剧集 | POST | `/api/v1/pgc/:pgc_id/episodes/add`，Body：集序、标题、vid 等 |
| 删除剧集 | DELETE | `/api/v1/pgc/:pgc_id/episodes/:id` |

- **类型**：1 番剧 / 2 纪录片 / 3 电影 / 4 电视剧（与后端枚举一致）。
- **剧集**：每集必填 `vid`（站内视频 ID）；创建 PGC 时至少提供一集。

具体请求/响应字段以实际后端 DTO 和 Swagger/文档为准。

## 5. 部署与访问

- **构建**：`npm run build`，产物输出到 `dist`（或配置的目录）。
- **部署**：将 `dist` 内容部署到 Go 服务的 `static/admin` 目录，或单独静态资源域名/路径；现有 Go 路由已支持 `r.Static("/admin", "./static/admin")`，访问 `/admin` 即可打开管理前端。
- **环境**：前端需可配置 API 基地址（如 `VITE_API_BASE`），请求统一发往当前环境的后端地址。

---

## 6. 开发顺序建议

1. **环境与框架**：初始化 Vue 3 + Vite 项目，接入 Element Plus，配置 API 基地址与 Axios（或 fetch）请求封装，统一带 Token。
2. **鉴权**：实现 Token 输入/存储与请求头注入；可选登录页或列表页顶栏输入。
3. **列表页**：PGC 列表、筛选、搜索、分页、操作按钮（编辑、剧集、删除）。
4. **创建页**：新建 PGC 表单与剧集表，提交调用 POST PGC 接口。
5. **编辑页**：拉取详情并回填表单，提交调用 PUT 接口。
6. **剧集管理页**：按 pgc_id 拉取剧集列表，添加/删除剧集，调用对应 episodes 接口。
7. **删除**：列表或详情删除，二次确认后调用 DELETE。
8. **扩展**：封面上传、草稿/发布、剧集排序等按需求排期。

---

## 7. 文档与约定

- **状态与枚举**：PGC 状态、类型等与后端保持一致，前端仅做展示与表单取值，不做业务枚举扩展。
- **错误处理**：接口失败时统一提示（如 Element Plus Message），必要时根据 code/msg 做友好文案。
- **本文档**：后续若 API 或功能有变更，请同步更新本文档，便于团队与后续开发使用。
